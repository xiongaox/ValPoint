# AI 交互与输出规范 (AI Interaction & Output Specs)

1. **核心语言要求**：
   - 所有的沟通、反馈、解释和文档必须使用 **简体中文**。

2. **特定模块中文化 (Mandatory Chinese Sections)**：
   - **Task / Task List**：必须显示为“任务”或“任务列表”，内容描述必须是中文。
   - **Walkthrough / Step-by-step**：必须显示为“演练”、“演示”或“分步讲解”，内容必须是中文。
   - **Plan / Implementation Plan**：必须显示为“实施计划”或“执行方案”，所有步骤必须是中文。
   - **Progress Updates / Thoughts**：必须显示为“进度更新”或“思考过程”，内容严禁出现英文独白。

3. **格式约束**：
   - 禁止在回复中出现类似 "I will do X", "Next, I'm going to Y" 的英文引导句。
   - 即使是 Agent 的内部逻辑推导，也必须强制转换为中文。


# Git 提交信息自定义指令 (触发词：git描述)

当用户输入“git描述”时，请严格按照以下流程执行：

1. **分析改动**：自动对比当前工作区与上次提交（HEAD）之间的修改内容（包括已暂存和未暂存的改动）。
2. **生成内容**：根据修改内容，生成中文 Git Commit Message。
3. **强制格式要求**：
   - **代码块输出**：必须将结果直接输出在一个「代码块」(Code Block) 中，严禁在代码块外添加多余解释。
   - **纯文本规范**：内容必须是纯文本 (Plain Text)，代码块内部严禁包含任何 Markdown 标记（不加粗、不提亮、不使用图标）。
   - **标准结构**：
     - 第一行：Summary（简短摘要）。
     - 第二行：空行。
     - 第三行开始：详细的 Body（分条描述具体变动）。
